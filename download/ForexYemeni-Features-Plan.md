## 🔐 2. قفل الجهاز (Device Fingerprint)

**الفكرة:** كل عميل يُسجل دخوله من جهاز، يتم حفظ بصمة الجهاز (Device ID). عند الدخول من جهاز جديد، يُطلب OTP إضافي.

**طريقة التنفيذ:**
- إنشاء `deviceId` فريد باستخدام `navigator.userAgent + screen resolution + timezone`
- حفظ `deviceId` في Firestore مع بيانات المستخدم
- عند تسجيل دخول: مقارنة `deviceId` الحالي بالمحفوظ
- لو مختلف → إرسال OTP إضافي للبريد/الرسالة مع رسالة "جهاز جديد"
- إضافة زر "تسجيل هذا الجهاز" بعد التحقق

**المكان في الكود:**
- دالة جديدة `_getDeviceId()` في بداية السكريبت
- تعديل دالة تسجيل الدخول `login()` — إضافة فحص الجهاز
- إضافة حقل `registeredDevices[]` في بيانات المستخدم

**الواجهة:**
- رسالة تنبيه عند دخول من جهاز جديد
- زر "تسجيل الجهاز" بعد إدخال OTP الثاني

## 📱 3. سجل الجلسات (إدارة الأجهزة)

**الفكرة:** صفحة في حساب العميل تعرض جميع الأجهزة المسجلة مع إمكانية حذف أي جهاز.

**طريقة التنفيذ:**
- حفظ مصفوفة `devices[]` لكل مستخدم: `{id, name, date, ip, active}`
- إنشاء عرض جديد `renderUserDevices(C)`
- زر حذف جهاز → يزيله من القائمة ويُسجل خروج الجهاز

**الواجهة:**
- أيقونة 📱 جديدة في الصفحة الشخصية
- قائمة بالأجهزة: الاسم، التاريخ، الحالة (نشط/غير نشط)
- زر حذف بجانب كل جهاز
- رسالة تأكيد قبل الحذف

## 💰 6. رسوم سحب ديناميكية

**الفكرة:** رسوم السحب تتغير حسب الشبكة المختارة (TRC20 أرخص من ERC20 من BEP20)

**طريقة التنفيذ:**
- إضافة إعدادات: `fee_trc20`, `fee_erc20`, `fee_bep20`
- عند اختيار شبكة السحب → يعرض الرسوم المناسبة
- حفظ الشبكة مع طلب السحب

**الواجهة:**
- في صفحة السحب: اختيار الشبكة (TRC20 / BEP20 / ERC20)
- عرض الرسوم بجانب كل شبكة
- تحديث المبلغ الصافي تلقائياً عند تغيير الشبكة

---

## 📊 7. صافي السحب (عرض قبل التأكيد)

**الفكرة:** قبل تأكيد السحب، يظهر للعميل تفصيل كامل: المبلغ المطلوب - الرسوم - العمولة = المبلغ الصافي

**طريقة التنفيذ:**
- حساب تلقائي: المبلغ - رسوم الإدارة - عمولة الشبكة
- عرض البطاقة التفصيلية قبل زر التأكيد

**الواجهة:**
```
┌─────────────────────────────┐
│ 💰 تفاصيل السحب            │
│                              │
│ المبلغ المطلوب:    100 USDT  │
│ رسوم الإدارة:      -2 USDT   │
│ رسوم الشبكة:       -1 USDT   │
│ ─────────────────────────── │
│ ✅ المبلغ الصافي: 97 USDT   │
│                              │
│ [تأكيد السحب]               │
└─────────────────────────────┘
```

## 📈 8. لوحة تحكم للعميل (رسوم بيانية)

**الفكرة:** صفحة رئيسية للعميل تعرض رصيده عبر الوقت برسم بياني + إحصائيات سريعة

**طريقة التنفيذ:**
- إنشاء عرض جديد `renderUserDashboard(C)`
- استخدام Chart.js (موجود مسبقاً في التطبيق)
- تجميع بيانات الرصيد من سجل العمليات `logs[]`

**الواجهة:**
- رسم بياني خطي للرصيد عبر 30 يوم
- بطاقات: إجمالي الإيداع، إجمالي السحب، الرصيد الحالي
- آخر 5 عمليات

## 📊 9. إحصائيات الإدارة المتقدمة

**الفكرة:** صفحة في لوحة الإدارة تعرض إحصائيات مفصلة بالرسوم البيانية

**طريقة التنفيذ:**
- إنشاء عرض جديد `renderAdminStats(C)`
- رسوم بيانية: إيداعات يومية، سحوبات يومية، صافي الأرباح
- إحصائيات: عدد المستخدمين النشطين، إجمالي الأرصدة، أكثر العملاء إيداعاً

**الواجهة:**
- تبويبات: يومي / أسبوعي / شهري
- رسم بياني شريطي للإيداعات والسحوبات
- رسم دائري لتوزيع العملات
- جدول أفضل العملاء

## 📄 10. تصدير البيانات (Excel + PDF)

**الفكرة:** إمكانية تصدير سجل العمليات والإيداعات والسحوبات كملف Excel أو PDF

**طريقة التنفيذ:**
- استخدام XLSX library (موجود مسبقاً) لـ Excel
- استخدام jsPDF (موجود مسبقاً) لـ PDF
- إنشاء دالة `exportData(type, format, data)`

**الواجهة:**
- زر "تصدير" في أعلى كل قائمة (الإيداعات، السحوبات، العمليات)
- اختيار: Excel 📊 أو PDF 📄
- يحمل الملف مباشرة

## ⭐ 11. نظام نقاط التجار

**الفكرة:** نظام مستويات للتجار حسب نشاطهم. كلما أكثر عملياته يرتقي مستوى

**طريقة التنفيذ:**
- مستويات: 🥉 برونزي (0-50 عملية) → 🥈 فضي (51-200) → 🥇 ذهبي (201-500) → 💎 ماسي (500+)
- إضافة حقل `merchantPoints` و `merchantLevel` لكل تاجر
- عمولة أقل لذوي المستوى الأعلى (حافز)

**الواجهة:**
- شريط التقدم في صفحة التاجر
- الأيقونة بجانب اسم التاجر (🥉🥈🥇💎)
- عرض المستوى والمتطلبات للترقي

## 🔗 12. رابط إحالة للتاجر

**الفكرة:** كل تاجر يحصل على رابط فريد. أي عميل يسجل من هذا الرابط يُنسب للتاجر تلقائياً

**طريقة التنفيذ:**
- إنشاء رابط: `https://app.com/?ref=MERCHANT_ID`
- حفظ `referredBy` عند تسجيل العميل الجديد
- تاجر يحصل على عمولة إضافية من إيداعات العملاء المحالين

**الواجهة:**
- في صفحة التاجر: "رابط الإحالة الخاص بك"
- زر نسخ الرابط
- عدد العملاء المحالين + إجمالي عمولاتهم

---

## 💳 13. حد ائتمان للتاجر

**الفكرة:** التاجر يحدد حد يومي للإيداعات. لو وصل الحد، يتوقف استقبال إيداعات جديدة

**طريقة التنفيذ:**
- إضافة حقل `merchantDailyLimit` في بيانات التاجر
- فحص الحد عند كل إيداع جديد
- إشعار للتاجر عند الوصول لـ 80% من الحد

**الواجهة:**
- في إعدادات التاجر: "الحد اليومي للإيداعات"
- شريط تقدم يظهر للمستخدم: "التاجر: 80/100 USDT اليوم"
- رسالة للعميل لو الحد اكتمل: "عذراً، وصل التاجر لحدوده اليومية"

## 🔔 18. إشعار صلاحية KYC

**الفكرة:** تنبيه للعميل والإدارة قبل انتهاء توثيق الهوية بـ 30 يوم

**طريقة التنفيذ:**
- إضافة حقل `kycExpiryDate` في وثيقة KYC
- فحص يومي: لو متبقي أقل من 30 يوم → إرسال إشعار
- إشعار على شكل بانر أصفر في صفحة العميل

**الواجهة:**
- بانر: "⚠️ توثيق الهوية ينتهي بعد 25 يوم - جدد الآن"
- زر "تجديد التوثيق"
- إشعار بريد إلكتروني

## 🚨 19. تنبيهات السحب الكبير

**الفكرة:** إشعار فوري للإدارة عند طلب سحب يتجاوز مبلغ معين

**طريقة التنفيذ:**
- إعداد: `large_withdrawal_threshold` (مثلاً 1000 USDT)
- عند سحب أكبر من الحد → إرسال Push + Email للإدارة
- عرض علامة ⚠️ في لوحة الإدارة

**الواجهة:**
- في الإعدادات: "حد تنبيه السحب الكبير"
- في قائمة السحوبات: علامة 🚨 بجانب السحب الكبير
- إشعار: "سحب كبير: العميل X يطلب سحب Y USDT"

## 📋 20. ملخص يومي للإدارة

**الفكرة:** رسالة/تقرير يومي تلقائي يتضمن ملخص: إيداعات + سحوبات + أرباح + مستخدمين جدد

**طريقة التنفيذ:**
- دالة `generateDailySummary()` تجمع بيانات اليوم
- إرسال كبريد إلكتروني يومياً في ساعة محددة
- حفظ آخر ملخص في Firestore للعرض الفوري

**الواجهة:**
- في لوحة الإدارة: بطاقة "ملخص اليوم"
```
┌─────────────────────────────┐
│ 📋 ملخص اليوم               │
│                              │
│ 📥 إيداعات: 15 (5,200 USDT) │
│ 📤 سحوبات: 8 (3,100 USDT)  │
│ 💰 صافي الأرباح: 450 USDT   │
│ 👥 مستخدمين جدد: 3          │
│ ⚠️ سحوبات معلقة: 2         │
└─────────────────────────────┘
```

## 🤖 22. ردود تلقائية للدعم (روبوت محادثة)

**الفكرة:** زر "المساعدة" يفتح نافذة محادثة مع روبوت يجاوب على الأسئلة الشائعة

**طريقة التنفيذ:**
- قاعدة أسئلة وأجوبة محددة مسبقاً (FAQ)
- روبوت يبحث في الأسئلة ويجيب بأقرب تطابق
- لو ما لقى الجواب → يحول للإدارة (يدوي)

**الواجهة:**
- زر 💬 "المساعدة" في الصفحة الرئيسية
- نافذة محادثة من الأسفل
- رسومات: فقاعات رسائل + مؤشر كتابة
- أزرار سريعة: "كيف أودع؟" / "كيف أسحب؟" / "توثيق الهوية" / "تواصل مع الإدارة"
