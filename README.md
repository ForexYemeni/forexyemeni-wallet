## 📖 وصف المشروع

**ForexYemeni Wallet** هي محفظة رقمية متكاملة مصممة خصيصاً لتلبية احتياجات السوق اليمني. توفر المنصة حلولاً شاملة لإدارة وتداول العملات الرقمية USDT مع دعم كامل للريال اليمني (YER).

المشروع مبني كتطبيق ويب متجاوب مع إمكانية تحويله إلى تطبيق أندرويد (APK) باستخدام **Capacitor**، مما يوفر تجربة سلسة على جميع الأجهزة.

### 🎯 ما الذي يجعل هذا المشروع مميزاً؟

- 💱 **تداول P2P مباشر** بين المستخدمين بدون وسيط
- 🔒 **أمان متعدد الطبقات** مع مصادقة ثنائية وحماية PIN
- 📊 **لوحة إدارة متكاملة** تضم أكثر من 17 تاب لمراقبة وإدارة كافة العمليات
- 📱 **تصميم متجاوب** يعمل على الويب والهواتف كتطبيق أصلي
- 🇾🇪 **مخصص للسوق اليمني** مع دعم USDT و YER

## 🚀 التثبيت والتشغيل

### 1. استنساخ المشروع

```bash
git clone https://github.com/your-username/forexyemeni-wallet.git
cd forexyemeni-wallet
```

### 2. تثبيت المكتبات

```bash
npm install
# أو باستخدام Bun
bun install
```

### 3. إعداد متغيرات البيئة

```bash
cp .env.example .env
```

### 4. ملء المتغيرات في ملف `.env`

```env
# Firebase Configuration
# أنشئ مفتاح Service Account من Firebase Console
FIREBASE_SERVICE_ACCOUNT={"type":"service_account","project_id":"your-project-id",...}

# Email Service (Google Apps Script - مجاني)
GOOGLE_APPS_SCRIPT_URL=https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec

# مفتاح سري لمصادقة الإيميلات (غيّره إلى أي نص عشوائي)
EMAIL_SECRET=your-random-secret-key
```

### 5. تشغيل المشروع

```bash
# وضع التطوير
npm run dev

# بناء المشروع
npm run build

# تشغيل في وضع الإنتاج
npm run start
```

سيكون التطبيق متاحاً على: **http://localhost:3000**

## 🔥 إعداد Firebase

### الخطوة 1: إنشاء مشروع Firebase

1. اذهب إلى [Firebase Console](https://console.firebase.google.com/)
2. اضغط على **"إضافة مشروع"**
3. أدخل اسم المشروع (مثال: `forexyemeni-wallet`)
4. اختر إنشاء مشروع جديد (بدون Google Analytics إن أردت)
5. اضغط **"إنشاء مشروع"**

### الخطوة 2: تفعيل Firestore

1. من القائمة الجانبية اختر **Firestore Database**
2. اضغط **"إنشاء قاعدة بيانات"**
3. اختر **"وضع الاختبار"** للبدء (يمكنك تغييره لاحقاً)
4. اختر موقع قاعدة البيانات الأقرب لمنطقة الخدمة

### الخطوة 3: الحصول على مفتاح Service Account

1. اذهب إلى **إعدادات المشروع** ⚙️
2. اختر تبويب **"حسابات الخدمة"**
3. اضغط **"إنشاء مفتاح خاص جديد"**
4. سيتم تنزيل ملف JSON - انسخ محتواه بالكامل
5. ضعه في ملف `.env` في المتغير `FIREBASE_SERVICE_ACCOUNT`

```json
FIREBASE_SERVICE_ACCOUNT={"type":"service_account","project_id":"your-project-id","private_key_id":"...","private_key":"...","client_email":"...","client_id":"...","auth_uri":"...","token_uri":"...","auth_provider_x509_cert_url":"...","client_x509_cert_url":"..."}
```

### الخطوة 4: إعداد قواعد الأمان (Security Rules)

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // قم بتعديل القواعد حسب احتياجاتك
    match /{document=**} {
      allow read, write: if false; // الوصول عبر Service Account فقط
    }
  }
}
```

## 📧 إعداد Google Apps Script (خدمة الإيميلات المجانية)

المشروع يستخدم Google Apps Script لإرسال البريد الإلكتروني مجاناً (بدون حاجة لخدمة مدفوعة مثل SendGrid).

### الخطوة 1: إنشاء السكربت

1. اذهب إلى [Google Apps Script](https://script.google.com/)
2. اضغط **"مشروع جديد"**
3. امسح الكود الافتراضي والصق الكود الموجود في ملف `GOOGLE_APPS_SCRIPT_CODE.gs`

### الخطوة 2: نشر السكربت

1. اضغط **"نشر"** ← **"نشر جديد"**
2. اختر نوع النشر: **"تطبيق ويب"**
3. في **"تنفيذ باسم"**: اختر **"أنا"**
4. في **"من يمكنه الوصول"**: اختر **"أي شخص"**
5. اضغط **"نشر"**
6. **منح الصلاحيات** عند الطلب
7. انسخ **رابط التطبيق** وضعه في `.env`:

```env
GOOGLE_APPS_SCRIPT_URL=https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec
```

### الخطوة 3: تأمين السكربت
